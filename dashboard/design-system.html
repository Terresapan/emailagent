Usage Guidelines

  • Primary (Lavender): Use for main actions, CTAs, and brand elements. The signature color of Neu-Editorial.
  • Accent (Teal): Use for highlights, secondary actions, and complementary elements.
  • Theme Colors: Use for backgrounds, surfaces, and structural elements.
  • Color Variants: Use dark/light/pale variants for hover states and depth.

Do / Don't

✓ Do

  • Use primary for main CTAs and important actions
  • Use accent for highlights and emphasis
  • Test color combinations for accessibility

✗ Don't

  • Use primary and accent together excessively
  • Use colors without sufficient contrast
  • Create new colors outside the palette

Accessibility Notes

  • Color Contrast: All text colors meet WCAG AA contrast requirements (4.5:1 for normal text, 3:1 for large text).
  • Color Independence: Don't rely on color alone to convey meaning. Use icons, labels, or patterns as well.
  • Focus States: Use primary color for focus indicators to ensure visibility.
  • Dark Mode: The palette is designed for dark backgrounds. Test on light backgrounds if needed.

Usage Guidelines

Do / Don't

✓ Do

  • Use one primary button per section
  • Group related actions together
  • Use destructive buttons sparingly

✗ Don't

  • Use multiple primary buttons in the same area
  • Make buttons too small to tap on mobile
  • Use destructive buttons for non-destructive actions

Accessibility Notes

  • Focus States: All buttons have visible focus states for keyboard navigation.
  • Color Contrast: Button text meets WCAG AA contrast requirements.
  • Touch Targets: Minimum 44x44px for mobile accessibility.
  • Screen Readers: Use descriptive button text or aria-labels.

✓ Do

  • Test spacing on different screen sizes

✗ Don't

Usage Guidelines

  • Success: Use for successful operations and confirmations.
  • Error: Use for errors, failures, and destructive actions.
  • Warning: Use for caution states and attention-grabbing elements.
  • Info: Use for informational messages and neutral feedback.
  • Duration: Auto-dismiss after 3 seconds for optimal UX.

Do / Don't

✓ Do

  • Use semantic colors for toast types
  • Keep messages concise and actionable
  • Position toasts consistently (usually bottom-right)

✗ Don't

  • Stack multiple toasts
  • Use toasts for non-action feedback
  • Make toasts persistent without user action

Accessibility Notes

  • Screen Readers: Toasts are announced by screen readers.
  • Focus Management: Ensure toasts don't trap keyboard focus.
  • Color Independence: Don't rely on color alone to convey meaning.
  • Dismissal: Allow users to dismiss toasts manually.
  • Timing: 3 seconds is optimal for reading without being intrusive.
